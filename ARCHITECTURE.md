## 1. Qt Desktop приложение (ЛОКАЛЬНО)

### Технологии:
- **C++17** + **Qt 6.5**
- **SQLite** - локальная база данных
- **QML** - пользовательский интерфейс
- **CMake** - система сборки

### Где работает:
✅ Windows компьютеры пользователей  
✅ Linux компьютеры пользователей  
✅ Работает ПОЛНОСТЬЮ ОФЛАЙН

### Функции:
- Управление группами и студентами
- Создание ведомостей
- Импорт из HTML портала
- Локальное хранение данных
- **Синхронизация с сервером** (когда есть интернет)

### Сборка:
```bash
cmake -S . -B build
cmake --build build --config Release
```

### Распространение:
- Установщик .exe для Windows
- AppImage для Linux
- Скачивается с GitHub Releases

## 2. Node.js API Server (RAILWAY.APP)

### Технологии:
- **Node.js 18+** + **Express**
- **PostgreSQL** - центральная база
- **JWT** - аутентификация
- **RESTful API** - HTTP эндпоинты

### Где работает:
☁️ **Railway.app cloud** (или любой другой Node.js хостинг)  
🌐 Доступен через интернет 24/7

### Функции:
- Прием данных от клиентов
- Хранение в центральной БД
- Раздача обновлений клиентам
- Аутентификация пользователей
- Разрешение конфликтов синхронизации

### Деплой:
```bash
cd server
npm install
npm start  # Локально для тестирования

# Или на Railway:
railway up
```

### API Endpoints:
```
POST   /api/auth/login          - Вход
POST   /api/auth/register       - Регистрация
GET    /api/groups              - Получить группы
POST   /api/groups              - Создать группу
GET    /api/students/:groupId   - Получить студентов
POST   /api/students            - Добавить студента
GET    /api/grade-sheets/:id    - Получить ведомость
```

## 3. Синхронизация данных

### Алгоритм:

```
Qt Client                     Railway Server
    │                              │
    │  1. POST /api/auth/login     │
    ├─────────────────────────────>│
    │                              │
    │  2. JWT Token                │
    │<─────────────────────────────┤
    │                              │
    │  3. GET /api/groups          │
    ├─────────────────────────────>│
    │                              │
    │  4. [список групп]           │
    │<─────────────────────────────┤
    │                              │
    │  5. Сравнить версии          │
    │     local vs server          │
    │                              │
    │  6. POST /api/groups         │
    │     (загрузить новые)        │
    ├─────────────────────────────>│
    │                              │
    │  7. Сохранить в SQLite       │
    │                              │
```

### Стратегии разрешения конфликтов:

**1. Last Write Wins (Последняя запись побеждает)**
```cpp
if (serverTimestamp > localTimestamp) {
    updateLocal(serverData);
} else {
    uploadToServer(localData);
}
```

**2. Version-based (По версиям)**
```cpp
if (serverVersion > localVersion) {
    updateLocal(serverData);
    localVersion = serverVersion;
}
```

**3. Manual Resolution (Ручное разрешение)**
```cpp
if (conflict) {
    showConflictDialog();
    userChooses(localData | serverData);
}
```

## 5. Workflow разработки

### Разработка Qt клиента:

```bash
# 1. Сборка
cmake -S . -B build
cmake --build build --config Debug

# 2. Запуск
build\Debug\offline-edm.exe

# 3. Тестирование
ctest --test-dir build -C Debug
```

### Разработка сервера:

```bash
# 1. Локальный запуск PostgreSQL
docker run -e POSTGRES_PASSWORD=pass -p 5432:5432 -d postgres

# 2. Настройка
cd server
cp .env.example .env
npm install

# 3. Миграции
npm run migrate

# 4. Запуск
npm run dev  # с автоперезагрузкой
```

### Деплой на Railway:

```bash
# Автоматически при push в GitHub
git add server/
git commit -m "Update API"
git push origin main

# Railway автоматически:
# - Обнаружит изменения
# - Пересоберет только server/
# - Задеплоит новую версию
```

## 6. Безопасность

### Qt Client:
✅ Локальные данные зашифрованы (EncryptionService)  
✅ Пароли хешируются с солью (bcrypt)

✅ JWT для аутентификации

## 9. FAQ

### Q: Нужен ли интернет для работы Qt приложения?
**A:** Нет! Приложение полностью работает офлайн. Синхронизация происходит только при наличии интернета.

### Q: Где хранятся данные?
**A:** Локально в SQLite (на компьютере пользователя) + централизованно в PostgreSQL (на сервере).

### Q: Что делать если нет интернета?
**A:** Приложение продолжает работать. Изменения сохраняются локально и синхронизируются при восстановлении связи.

### Q: Сколько стоит сервер?
**A:** Railway предоставляет $5/месяц бесплатно. Ваше приложение уложится в этот лимит.

### Q: Можно ли использовать другой хостинг?
**A:** Да! Сервер - это обычный Node.js + PostgreSQL. Можно деплоить на:
- Heroku
- DigitalOcean
- AWS
- Собственный сервер
