# Автоматическая синхронизация с порталом

## Описание

Система автоматической синхронизации с порталом НовГУ позволяет автоматически загружать и обновлять данные о группах и студентах без ручного вмешательства.

## Основные возможности

### 1. Фильтрация активных групп

При автоматической синхронизации импортируются **только активные группы текущего семестра**:

- ✅ Группы, которые учатся в текущем учебном году
- ✅ Группы с активными студентами (статус "обучается")
- ❌ Выпустившиеся группы (превысили срок обучения)
- ❌ Группы без активных студентов

#### Алгоритм определения активности группы

```cpp
bool isActiveInCurrentSemester() {
    // 1. Определяем текущий учебный год (сентябрь-август)
    int academicYear = currentYear;
    if (currentMonth < 9) academicYear--;
    
    // 2. Вычисляем, сколько лет группа учится
    int yearsStudying = academicYear - admissionYear;
    
    // 3. Определяем максимальный срок обучения:
    //    - Бакалавриат очно: 4 года
    //    - Бакалавриат заочно: 5 лет
    //    - Специалитет очно: 5 лет
    //    - Специалитет заочно: 6 лет
    //    - Магистратура очно: 2 года
    //    - Магистратура заочно: 3 года
    
    // 4. Группа активна, если не превышен срок обучения
    return yearsStudying >= 0 && yearsStudying < maxYears;
}
```

### 2. Периодическое обновление

При включенной автозагрузке система автоматически обновляет данные:

- **Частота**: раз в 24 часа
- **Условия**: 
  - Включена опция "Автозагрузка с портала при входе"
  - Есть сохранённая сессия портала
  - Есть подключение к интернету
- **Процесс**: 
  - Таймер запускается при открытии сессии пользователя
  - Синхронизация происходит в фоновом режиме
  - Обновляются только активные группы

### 3. Синхронизация при старте

При входе в приложение (если включена автозагрузка):

1. Проверяется наличие сохранённой сессии портала
2. Проверяется подключение к интернету
3. Автоматически загружаются данные с портала
4. Импортируются только активные группы текущего семестра

## Настройка

### В интерфейсе приложения

1. Откройте **Настройки портала** (кнопка "⚙️ Настройки портала" в боковом меню)
2. Введите **URL списка групп** (скопируйте из браузера после входа на портал)
3. Введите **логин и пароль** портала
4. Включите **"Запомнить сессию портала"**
5. Нажмите **"Войти и загрузить группы"**
6. После успешного входа включите **"Автозагрузка с портала при входе в приложение"**

### Программно

```cpp
// Включить автоматическую синхронизацию
appController->setPortalOnlineAutoSync(true);

// Установить URL списка групп
appController->setPortalGroupsIndexUrl("https://portal.novsu.ru/...");

// Запустить синхронизацию
appController->syncPortalOnlineSavedSession();
```

## Технические детали

### Файлы

- `src/portal/PortalTypes.h/cpp` - структуры данных и логика фильтрации
- `src/portal/PortalImportService.h/cpp` - сервис синхронизации
- `src/app/AppController.h/cpp` - контроллер приложения с таймерами

### Методы

#### `PortalGroupDetail::isActiveInCurrentSemester()`
Проверяет, является ли группа активной в текущем семестре на основе:
- Года поступления
- Курса
- Формы обучения (очная/заочная)
- Направления (бакалавриат/специалитет/магистратура)

#### `PortalGroupDetail::hasActiveStudents()`
Проверяет наличие студентов со статусом "обучается" или "СТ"

#### `AppController::schedulePeriodicSync()`
Запускает таймер для периодической синхронизации (24 часа)

#### `AppController::onPeriodicSyncTimer()`
Обработчик таймера - запускает синхронизацию при наличии интернета и сохранённой сессии

### Настройки

Настройки сохраняются в `QSettings`:

```cpp
settings.setValue("portal/onlineAutoSync", true);
settings.setValue("portal/groupsIndexUrl", "...");
settings.setValue("portal/savedLogin", "...");
```

```cpp
// В C++

    // ...

        scheduleOnlinePortalSync();  // Синхронизация при старте
```

### Пример 3: Фильтрация групп

```cpp
// В PortalImportService::applyGroupDetail()
if (m_filterActiveOnly) {
    if (!detail.isActiveInCurrentSemester()) {
        return false; // Пропускаем неактивную группу
    }
    
    if (!detail.hasActiveStudents()) {
        return false; // Пропускаем группу без активных студентов
    }
}
```

## Преимущества

1. **Актуальность данных** - автоматическое обновление раз в день
2. **Экономия трафика** - импортируются только активные группы
3. **Удобство** - не нужно вручную обновлять данные
4. **Безопасность** - пароль не сохраняется, используются cookies сессии
5. **Оффлайн-режим** - данные доступны без интернета после первой синхронизации

## Ограничения

1. Требуется первичный вход с логином и паролем
2. Сессия портала может истечь (требуется повторный вход)
3. Синхронизация работает только при наличии интернета
4. Фильтрация активных групп может пропустить группы с нестандартными сроками обучения

## Устранение проблем

### Проблема: "Сессия портала истекла"

**Решение**: Войдите заново с логином и паролем, включив "Запомнить сессию"

### Проблема: "Не найдены ссылки на группы"

**Решение**: Убедитесь, что URL скопирован правильно - это должна быть страница со списком ваших групп после входа на портал

### Проблема: "Группа не импортируется"

**Решение**: Проверьте, что группа активна в текущем семестре и имеет активных студентов. Если нужно импортировать все группы, отключите фильтр:

```cpp
m_portalImport->setFilterActiveOnly(false);
```
