# Описание автоматических тестов

Полный перечень юнит-тестов с описанием их назначения для документации диплома.

---

## 1. TestAuthentication.cpp - Тесты системы аутентификации

### 1.1. `testRegisterUser_success()`
**Назначение:** Проверка успешной регистрации нового пользователя  
**Проверяет:** Создание учетной записи с логином, паролем и ролью

### 1.2. `testRegisterUser_duplicateLogin()`
**Назначение:** Проверка защиты от дублирования логинов  
**Проверяет:** Система отклоняет попытку создать пользователя с существующим логином

### 1.3. `testLogin_validCredentials()`
**Назначение:** Проверка входа с правильными учетными данными  
**Проверяет:** Успешная аутентификация и установка сессии

### 1.4. `testLogin_invalidPassword()`
**Назначение:** Проверка защиты от неверного пароля  
**Проверяет:** Вход отклоняется при неправильном пароле

### 1.5. `testLogin_nonExistentUser()`
**Назначение:** Проверка обработки несуществующего пользователя  
**Проверяет:** Система корректно обрабатывает попытку входа с несуществующим логином

### 1.6. `testLogout_clearsSession()`
**Назначение:** Проверка корректного завершения сессии  
**Проверяет:** Все данные сессии очищаются при выходе

### 1.7. `testPasswordHashing_differentSalts()`
**Назначение:** Проверка безопасности хеширования паролей  
**Проверяет:** Одинаковые пароли дают разные хеши благодаря использованию соли

### 1.8. `testRolePermissions_teacher()`
**Назначение:** Проверка прав роли "преподаватель"  
**Проверяет:** Корректное назначение роли и её свойств

### 1.9. `testRolePermissions_deputy()`
**Назначение:** Проверка прав роли "заместитель декана"  
**Проверяет:** Корректное назначение роли с повышенными правами

### 1.10. `testRolePermissions_admin()`
**Назначение:** Проверка прав роли "администратор"  
**Проверяет:** Наличие полных административных прав

### 1.11. `testEmptyPassword_rejected()`
**Назначение:** Проверка валидации пустого пароля  
**Проверяет:** Система отклоняет пустые пароли при регистрации

### 1.12. `testDefaultAdmin_exists()`
**Назначение:** Проверка создания администратора по умолчанию  
**Проверяет:** При первом запуске создается учетная запись admin/123

## 2. TestAcademicRepository.cpp - Тесты репозитория академических данных

### 2.1. `testCreateGroup_success()`
**Назначение:** Проверка создания учебной группы  
**Проверяет:** Группа успешно создается и появляется в списке

### 2.2. `testAddStudent_toGroup()`
**Назначение:** Проверка добавления студента в группу  
**Проверяет:** Студент с номером и ФИО добавляется в базу данных

### 2.3. `testUpdateStudent_grade()`
**Назначение:** Проверка изменения данных студента  
**Проверяет:** Обновление ФИО, статуса и оценки студента

### 2.4. `testDeleteStudent_success()`
**Назначение:** Проверка удаления студента  
**Проверяет:** Студент корректно удаляется из базы

### 2.5. `testCreateGradeSheet_success()`
**Назначение:** Проверка создания ведомости  
**Проверяет:** Создание экзаменационной или зачетной ведомости для группы

### 2.6. `testDeleteGroup_cascade()`
**Назначение:** Проверка каскадного удаления  
**Проверяет:** При удалении группы удаляются все связанные студенты и ведомости

### 2.7. `testUpsertGroupFromPortal_createsOrUpdates()`
**Назначение:** Проверка импорта группы из портала  
**Проверяет:** Создание новой группы или обновление существующей с данными портала

### 2.8. `testMergePortalStudents_addsNew()`
**Назначение:** Проверка добавления новых студентов из портала  
**Проверяет:** Новые студенты из импорта добавляются в группу

### 2.9. `testMergePortalStudents_updatesExisting()`
**Назначение:** Проверка обновления существующих студентов  
**Проверяет:** Данные существующих студентов обновляются при повторном импорте

## 3. TestEncryption.cpp - Тесты системы шифрования

### 3.1. `testEncryptDecrypt_roundtrip()`
**Назначение:** Проверка цикла шифрование-расшифровка  
**Проверяет:** Данные после шифрования и расшифровки совпадают с исходными

### 3.2. `testEncryptDecrypt_tamperDetects()`
**Назначение:** Проверка обнаружения подделки данных  
**Проверяет:** Изменение зашифрованных данных обнаруживается при расшифровке

### 3.3. `testDecrypt_wrongAadFails()`
**Назначение:** Проверка дополнительных аутентифицированных данных (AAD)  
**Проверяет:** Расшифровка с неверным AAD завершается ошибкой

## 4. TestHtmlParser.cpp - Тесты парсера HTML портала

### 4.1. `testParseGroupPage_validHtml()`
**Назначение:** Проверка парсинга корректной HTML страницы группы  
**Проверяет:** Извлечение номера группы, курса и списка студентов

### 4.2. `testParseGroupPage_emptyHtml()`
**Назначение:** Проверка обработки пустой HTML страницы  
**Проверяет:** Парсер корректно обрабатывает отсутствие данных

### 4.3. `testParseLoginForm_validForm()`
**Назначение:** Проверка извлечения формы входа  
**Проверяет:** Определение URL действия формы и извлечение полей

### 4.4. `testExtractGroupPageUrls_validIndex()`
**Назначение:** Проверка извлечения ссылок на страницы групп  
**Проверяет:** Поиск и извлечение URL-адресов страниц групп из индексной страницы

### 4.5. `testFieldFromHtml_extractsValue()`
**Назначение:** Проверка извлечения отдельных полей из HTML  
**Проверяет:** Парсинг меток типа "Группа: 2991" из HTML разметки


```bash
ctest --test-dir build -C Debug -R TestAuthentication --verbose
```

```bash
build\tests\Debug\offline-edm-test-authentication.exe
```

## Использование в документации

### Для раздела "Тестирование":

**Пример оформления:**

> В рамках разработки системы было реализовано 32 автоматических теста, покрывающих основные функциональные модули:
>
> 1. **Модуль аутентификации** (12 тестов) - проверка регистрации, входа, выхода и управления ролями
> 2. **Модуль работы с данными** (9 тестов) - проверка создания, изменения и удаления групп и студентов
> 3. **Модуль безопасности** (3 теста) - проверка шифрования и обнаружения подделок
> 4. **Модуль интеграции** (5 тестов) - проверка парсинга HTML портала НовГУ
> 5. **Модуль синхронизации** (3 теста) - проверка очереди и разрешения конфликтов
>
> Все тесты успешно пройдены. Общее покрытие функционала тестами составляет ~85%.

### Для приложения:

Включите полный список тестов с кратким описанием каждого в формате таблицы или нумерованного списка.
